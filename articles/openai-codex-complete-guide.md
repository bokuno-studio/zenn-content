## Codex デスクトップアプリの主要機能

動画冒頭で紹介されていた機能一覧がこちらです。

![Codex アプリの機能一覧とデモ画面（0分頃）](../images/codex-v1-01.jpg)

- **複数コーディングエージェントの並列実行** — 別々のプロジェクトスレッドで複数エージェントを同時稼働
- **Isolated Worktrees** — 同じリポジトリ上で複数エージェントが競合なく作業可能

- **インアプリ画像生成** — アプリ内でそのまま画像を生成・精製
- **インブラウザコメント** — 実装されたページに直接コメント可能
- **GitHub PR レビュー・Jira 連携** などの開発ワークフロータスク

## Skills（スキル）と Plugins（プラグイン）の違い

動画中盤では Excalidraw スキルを使って構造図が自動生成されました。

![Skill と Plugin の構造を Excalidraw で図解（10分頃）](../images/codex-v1-02.jpg)

| | Skill（スキル） | Plugin（プラグイン） |
|---|---|---|
| **定義** | 特定タスク向けの再利用可能なワークフローパッケージ | Codex に追加機能をインストールする単位 |
| **役割** | 指示・リソース・スクリプトをまとめて Codex のタスク処理能力を拡張 | スキル・アプリ・MCP Server・統合機能をバンドル |
| **目的** | ワークフローを Codex に確実に実行させるレシピ | 接続システムとパッケージツールへのアクセスを提供 |

:::message
**シンプルな覚え方:**
- Skill = 再利用可能なレシピ
- Plugin = そのレシピを Codex に持ち込めるインストール可能パッケージ
:::

## Automations（自動化）機能

![Automation の構成図 — Custom と Built-in の2種類（1時間10分頃）](../images/codex-v1-04.jpg)

### 2 種類の Automation

**カスタム Automation**
- 例: **YouTube Researcher**（Supadata API を使用）
  - YouTube 動画のトランスクリプトを取得（月 100 件まで無料）
  - 調査結果を自動で Word 文書にまとめて出力
- 外部 API キーを設定することで任意のサービスと連携

**ビルトイン Automation**
- Word Doc など標準で用意されているもの
- 追加設定なしで利用可能

### 実践的なユースケース

```
「先週の競合リサーチをまとめて Word にして」
→ Supadata で YouTube を調査 → 結果を Word 文書に自動出力
```

Codex デスクトップアプリは **コーディングだけでなく、デザイン・ドキュメント・調査・自動化** まで対応できる総合 AI エージェントプラットフォームです。

タスクの規模や精度要件に合わせてモデルや処理負荷を選択できる点も Codex の強みです。

スキルとは特定の種類のタスクのための再利用可能なパッケージです。スキルは「再利用可能なレシピ」、プラグインは「Codex に機能を追加するインストール可能な単位」です。

Claude アプリの「カスタマイズ」タブを見ると「スキル」と「コネクター」があります。OpenAI には「プラグイン」と「スキル」があります。ツールによって用語が異なりますが、私はこれらをまとめて「スキル」と考えています。スキルはモデルの能力を拡張する方法です。

「メールを確認して今日届いた緊急メッセージを教えて」と言えるようにするには、Codex にメールのスキルを与える必要があります。Google カレンダープラグインをインストールする例を見てみましょう。インストールするとブラウザでサインインして「続行」→「すべて選択」→「続行」とクリックします。確認後 Google カレンダーが接続されます。

デザインが出来上がってきました。テクノロジー・パフォーマンス・ローンチと書かれたトップバーが作成されました。ボタンが少し重なっているようですが、AI が気づいて修正するはずです。

これは新機能を説明するいい機会です。多くの AI ツールには「メッセージキューイング」があり、AI が作業中にプロンプトを入力しても完了するまで待ちます。しかし Codex には「ステアリング」機能があります。作業中にスクリーンショットを貼り付けて「重なりがあるので修正してください」と言うと、Enter を押すとデフォルトでキューに入れられます。「Steer」をクリックすると作業中でも即座に入力できます。最新のツール呼び出しが終わると同時にプロンプトが送信されます。

Paper は AI エージェント（Codex、Claude Code など）と接続するために設計されたツールです。Figma の統合と比べて Paper の方がずっと楽しいです。「4つのバリエーションを作ってください」と言うと、異なる背景色で4つのバリエーションが並行して作成されます。

これからプラグインはどんどん増えていくでしょう。世界中のすべてのソフトウェアプラットフォームが OpenAI の公式プラグインリストのトップに掲載されたいと思うようになるはずです。GitHub、Linear などのツールが並んでいるこのリストに表示されることは非常に価値があります。

独自のスキルの作成方法を説明しましょう。私が繰り返しやっていたことの一つは YouTube のトランスクリプトを取得することでした。API があれば便利だと思いました。

API というのは、技術的でない方向けに説明すると、他のソフトウェアから特定の情報をコードでアクセスする方法です。バイブコーディングで OpenAI のチャットをアプリに追加する際に OpenAI API キーが必要なのと同じ概念です。

「YouTube のトランスクリプトを取得できる API のトップ5を教えてください」と聞くと、SuperData、Transcript API、YouTube Transcript.io などが一覧表示されました。SuperData を使ってみることにしました。


**[30:00]**

SuperData は月100件無料でトランスクリプトを取得できます。API キーをコピーして Codex に戻ります。「特定の YouTube チャンネルの最新10本の動画を要約できるスキルを作ってください。SuperData API を使ってすぐに取得できるようにしてください。スキルを作成してください」と指示します。

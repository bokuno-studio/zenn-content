## はじめに

Anthropic CEO の Dario Amodei が Dwarkesh Patel の Podcast に再登場。前回対談から3年を経て、AI 技術の進展・エージェント経済の到来・AGI ガバナンスの難題まで2時間超にわたり深掘りした対話です。

## 1. AI技術の進展：予測との比較

![Dario Amodeiが語るAI技術の進展](/images/screenshot-dario-01.jpg)

> 「技術の指数関数的成長は、おおむね予測通りに進んでいる」

**RL スケーリングの発見**が最大の変化です。事前学習と同様のスケーリング則が RL（強化学習）でも成立することが確認されました：数学コンテスト（AIME）その他の RL タスクで、学習時間に対して線形にスコアが向上するという結果が各社から報告されています。

**2035年までの確信度は90%**。唯一の不確実性は「検証不可能なタスク」（火星ミッション計画・CRISPR 発見のような根本的科学的発見・小説執筆）にあるとしています。

### 人間の学習とモデルの学習の比較

LLM の学習には人間に対応するものがない独自のフェーズがあります：

| フェーズ | 人間 | LLM |
|---------|------|-----|
| 長期学習 | 数ヶ月〜年 | 事前学習（事後は固定） |
| 短期学習 | 数時間〜日 | In-context learning |
| 反射 | ミリ秒 | 推論（inference） |
| 継続学習 | 毎日の経験蓄積 | **現在は存在しない** |

## 2. Claude Code とエージェントの現実

![Dario Amodeiが語るClaudeとエージェント](/images/screenshot-dario-02.jpg)

> 「ループを閉じる能力——自己完結したシステムでのフィードバックサイクル——が、どれほどの生産性向上をもたらすかを考えさせられる」

Claude Code は大企業でも個人開発者と同じスピードで導入できるはずですが、実際の普及は組織の複雑さによって遅れています。Dario は「技術の普及は以前の技術より速い。しかし無限に速いわけではない」と強調します。

---

## 3. 「天才の国」という概念

> 「データセンターに天才が集まった国に匹敵する計算能力を詰め込む。それが今後の形だ」

Dario の「天才の国（Country of Geniuses）」：多数の賢いエージェントが並列で動くシステム。継続学習がなくても、事前学習の汎化と in-context learning の組み合わせで到達できる可能性がある。

## 4. AI経済学：コストと収益の均衡

> 「各モデルは利益を生んでいる。しかし会社は損している。次の訓練に指数関数的な投資が続くから」

推論の粗利は50%超。しかし次世代モデル訓練への指数的投資が続くため各社は赤字。均衡はスケールアップが安定化したとき。

---

## 5. バイオリスク・ガバナンス・Anthropic文化

最も重要な安全懸念はバイオテロリズム。「今年後半に深刻と判断する世界が来るかもしれない」。民主主義国家の連合が交渉してルールを設定すべき。Anthropic は AI コンスティテューションを公開し、全社タウンホールを隔週開催。

## 📝 動画全文書き起こし（日本語訳）

**[0:00]**
**D:** 3年前に話した。この3年間で最も大きかった変化は何ですか？
**Da:** 技術の指数関数的成長はおおむね予測通りです。コーディングという具体的な方向には予測していませんでしたが、指数的成長そのものは見た通りです——賢い高校生から賢い大学生、そして博士課程・専門職レベルへという進歩は。最も驚いているのは、指数関数の終わりにどれだけ近づいているかが公的に認識されていないことです。それは本当に信じられない。バブルの内外を問わず、世の中の人々はいつもの政治的な話ばかりしている。この指数関数の終わりに差し掛かっているのに。スケーリングについての話をしたい。3年前と同じ質問ですが、今はもっと複雑な問いになっている。3年前は事前学習スケーリング則という公知の指標があったが、今は RL スケーリングがあって公的なスケーリング則がない。仮説は何なのか？

**Da:** 私は 2017 年から同じ仮説を持っています——「大きな計算の塊（Big Blob of Compute）仮説」と呼んでいたものです。GPT-1 が出たばかりのころに書いたドキュメントで、言語モデルに特化したものではなく、ロボティクスや強化学習なども含めた一般的なものでした。仮説の要点は、「賢い手法や新手法はあまり重要ではない。重要なのは7つのことだけ」というもの：①生のコンピュート量、②データ量、③データの質と分布（幅広い分布が必要）、④学習時間、⑤スケールできる目的関数（事前学習目的関数と RL 目的関数の両方がそれにあたる）、⑥⑦正規化・数値安定性。この仮説を今も変えていません。事前学習スケーリング則はその一例。そして今、RL でも同じことが見えています。RL フェーズも事前学習と同様にスケールしている——他社も数学コンテスト（AIME）などで、学習時間に対してスコアが線形に伸びることを公開しています。それは幅広い RL タスクで見られます。

**[0:05]**
**D:** RL スケーリングについて。Ilya Sutskever を去年インタビューした。彼は LLM に非常に懐疑的だった。彼の反論を言い換えると、「真のコア人間学習能力を持つものは、何十億ドルものデータとコンピュートや特殊環境なしに Excel の使い方を学べるはず。それが必要なことは、その中核的な学習アルゴリズムが欠如していることを示す。間違ったものをスケールしている」というものだ。なぜそんなにスケールするのか？

**Da:** いくつかのことを混同しています。RL が事前学習と本質的に違うとは思いません。事前学習を見ると、GPT-1 以前のモデルはファンフィクションや文学テキストなど狭い分布で訓練されていました。そして GPT-2 でインターネット全体のスクレイプ（Common Crawl 等）で訓練したとき初めて汎化が起きた。RL でも同じことが起きていると思います——最初は数学コンテストという単純なタスク、次にコードを含む幅広いタスク、そしてさらに多様なタスクへ。汎化が増えていく。これで「RL vs 事前学習」の部分は取り除けます。でも謎はある——事前学習では何兆ものトークンを使う。人間はそれほど言葉を見ない。サンプル効率が違う。モデルはゼロから始まって、はるかに多くの訓練が必要。しかし、訓練後に長いコンテキストを与えると、そのコンテキスト内での学習と適応が非常に優れている。思うに、事前学習は人間の学習と人間の進化の中間にあるものかもしれない。人間の脳はブランクスレートではなく、進化で形成された領域が接続されている。言語モデルはもっとブランクスレートに近い。ランダムな重みから始まる。だから事前学習は...

**[0:10]**
**Da:** ...人間の進化と人間のリアルタイム学習の中間空間に存在する。in-context learning は長期学習と短期学習の中間。だから、進化→長期学習→短期学習→瞬間反射という人間の階層の中で、LLM フェーズはその階層上に存在するが、人間のモードと完全に一致する点にあるわけではない。

**D:** 少し混乱している。たとえば進化のアナロジーを使うなら、in-context learning から超サンプル効率な学習が得られるなら、なぜ RL 環境企業が「この API の使い方」「Slack の使い方」などを教えているのか？もしすぐに習得できるエージェントが出現するなら、なぜそこに重点を置くのか？

**Da:** 目標はすべてのスキルをモデルに教えることではない。事前学習でインターネット全体のすべての文書を網羅しようとしていないのと同様。GPT-1 から GPT-2 への転換、つまり汎化に到達した瞬間を間近で見た。モデルに家の価格と面積のリストを与えると線形回帰をやる——一度も見ていないのに。RL 環境でも同様です——広い分布のデータを使って汎化を得ようとしている。議論の核心は「10年以内に AGI」か「まだ先か」という話だと思う。私は今、90% の確信で10年以内と言える。2035年までに起きないと言う方が非主流になると思う。なぜ1年後で10年後ではないのか？

**[0:15]**
**Da:** 二つの主張があります。弱い主張と強い主張。弱い主張から始めると——2019年にスケーリングを初めて見たとき、これが機能するかは半々だった。「誰も考えないが、私はこれが起こる確率50%だと思う」と。今、「データセンターに天才の国」という基本的仮説については90%の確信があります。それ以上は言いにくい——世界は予測不能なので。不確実性の残り部分は、コーディングのような検証可能なタスクについては例外的な理由でなければ1〜2年以内に到達する確信がある。検証困難なタスク——火星ミッション計画、CRISPR のような根本的科学的発見、小説執筆——についてはわずかな不確実性が残る。ほぼ確実に到達する経路があるが。10年スパンでは90%。これは言える最大の確信度に近い。

**D:** 検証の強調は、モデルが人間の真の学習に対応するものを持つという信念の欠如を示唆しているように見える。

**Da:** 検証可能なことから検証不可能なことへの汎化はすでに見えている。世界が「到達しない」という世界は、検証可能なことはすべてできるが検証不可能なことへの汎化が不完全という世界。しかしソフトウェアエンジニアリングでさえ設計文書などが必要で、モデルはそれも悪くない。より弱い主張で言えば、ソフトウェアエンジニアリングにはすでにほぼ到達している。AIが書くコードの行数という指標では、すでに90%に近い場所にある企業もある。しかし「書かれた行数」と「生産性向上の大きさ」は別の話。

**[0:20]**
**D:** Claude Code を見るたびに思う——自己完結したシステムでループを閉じることが、どれほどの広範な生産性向上をもたらすか。天才の国の推定に影響するかもしれない。

**Da:** 同意します——これがすぐには起きない理由でもある。同時に、効果は非常に速い。二つの極がある。一方は「AI は経済に拡散するまで時間がかかる」という緩やかな普及論。他方は「指数関数上に指数関数を書けば Dyson 球ができる」という再帰的自己改善論。Anthropic で見たのは奇妙な年10倍の売上成長——2023年に0→1億ドル、2024年に1億→10億ドル、2025年に10億→90〜100億ドル。その曲線は今年も続いており、1月だけで数十億を追加した。もちろんこれは GDP の大きさで頭打ちになるが、速い。そして速いが無限ではない。経済拡散にはループを閉じること、変更管理が必要。「古いソフトウェアがコンパイル前にモデルをチェックする——それを書き直す必要があるが、モデルに指示しなければ始まらない」というフィクションを理解する必要がある。すべての証拠は「能力の高速な指数関数」と「それを追う経済拡散の別の高速な指数関数——無限ではないが以前のどの技術より速い」という二つの指数関数と整合しています。

**D:** 「拡散」は「モデルが何かを実際にはできない」ことを隠すための言い訳として使われているように感じる。AI には人間より統合しやすいはずの利点がある——Slack 全体を読める、同じコピーが知識を共有できる、採用の問題がない。なぜ拡散が現実のボトルネックなのか？

**[0:25]**
**Da:** 拡散は非常にリアルです。AI が以前の技術より速く拡散するという主張で、ゆっくりという意味ではない。例として Claude Code がある——設定は非常に簡単で、個人開発者もスタートアップも大企業の開発者も技術的には同じスピードで導入できるはず。そして大企業に Claude Code を販売しており、大企業も通常より速く採用している。しかし時間がかかる——法務を通過し、全員にプロビジョニングし、セキュリティとコンプライアンスを通過し、会社のリーダーが「これが生産性向上になる」と説明し、3000人の開発者へのロールアウトを計画する。この会話を毎日している。年10倍ではなく20〜30倍にしようとしているが、たとえ AGI や「データセンターの天才の国」が来ても、無限に魅力的な製品にはならない。説得力はある——年3〜5〜10倍の成長が数千億ドル規模でも起きるかもしれない。それは歴史上例がないが、無限ではない。私が言いたいのは「拡散があるから大したことない」ではなく、本当に天才の国があれば全員が知る——ここにいる全員が、ワシントンの人々も。今はまだそこにない。

**[0:30]**
**D:** 3年前の予測——1時間話せば一般的に論理的な人間と区別がつかないモデル——は当たったと思う。しかし私の内部期待は、そういうシステムがホワイトカラーの仕事の多くを自動化できるというものだった。具体的な能力について話しましょう。例えばビデオエディターの場合——数ヶ月かけて私の好みや視聴者の好みを学び、6ヶ月後に固有の能力を持つ。そういう AI システムはいつ来ると思うか？

**Da:** ビデオエディターの例でいえば、天才の国がそれをできるようになる。方法は——コンピュータ画面を自由に操作し、過去のインタビューを見て、Twitter の反応を見て、あなたのスタッフと話し、編集履歴を見て——それから仕事をする。これはいくつかのことにかかっている。一つはコンピュータユースの信頼性。OSWorld のベンチマークで私たちがコンピュータユースを最初にリリースしたときは15%程度だったが、今は65〜70%になった。しかし信頼性が一定の閾値を超える必要がある。それ以外には学習の問題がある。私でさえ、テキストイン/テキストアウトのタスクで、7/10 の仕事はしてくれるが、人間の従業員のように継続的に関与して改善させる方法がないから、まだ人間を雇っている。

**Da:** でもコーディングエージェントについて言えば、学習を職場でやることが妨げているとは思わない。Anthropic 内部にはコードを一行も書かない開発者がいる。GPU カーネル、チップ——以前は自分で書いていたが、今は Claude にやらせている。コードベースへの親しみや「会社で1年働いたような感覚」は不満リストの上位に来ない。コーディングでは外部に存在する「コードベース」という記憶の足場があり、それが in-context で読める。他の仕事にはそれがない。

**[0:35]**
**D:** 「コードベースを読み込むだけで6ヶ月の学習に相当する」というのは興味深い。しかし量的研究では、経験豊富な開発者が主観的には生産性向上を感じたが、実際のマージ量は20%下がっていたという結果がある。定性的な感覚と客観的な指標の乖離をどう説明するか？

**Da:** Anthropic 内部では非常に明確。私たちは商業的プレッシャー下にあり、安全への取り組みという追加の重荷もある。売上10倍の曲線を維持しながら価値観を保つ圧力は巨大。そこに偽の生産性を感じているヒマはない。これらのツールが生産性を下げているなら、競合他社がツールを使うことを心配しない。数ヶ月ごとにモデルリリースという形の世界の終わりを見ており、自分を欺く余裕はない。

それより気になるのは「なぜ最良のコーディングモデルを持つ会社が長期的な優位性を保てないのか」という問い。私のモデルでは、生産性向上は段階的に増えている——6ヶ月前は全要素生産性で5%程度、今は15〜20%。5%では誰も気づかないが、15〜20%は重要になり始めている。複数の会社が私たちのモデルを使っているので、「最良のモデルを持つ会社が最も恩恵を受ける」という完全な分離もない。全体として「ソフトスタート」モデルと整合している——急激ではなく、滑らかな指数関数。

**[0:40]**
**D:** 継続学習がなければ世界変革は難しいと思っている。6ヶ月働いた人間が最大限の力になるなら、AI がそれをできないのであれば。

**Da:** 二つのことを分けて考えます。まず現在の技術の状態——事前学習と RL の2段階があり、大量のデータとコンピュートを使う。ブランクスレートから始まって多くの訓練が必要だが、一度学習すれば広い知識を持つ。日本のサムライの歴史について私より詳しく、野球についても、ローパスフィルターについても。その広い知識は in-context learning と組み合わさる——100万トークンのコンテキストに何日もかけて読む量と同等の内容を詰め込める。これだけで、天才の国の多くの部分に到達できるかもしれない。ギャップはあるかもしれないが、この既存のパラダイムだけで数兆ドルの収益を生み出せると思う。

二つ目は継続学習のアイデア。これも1〜2年で解決できる可能性があると思っている。取り組んでいるし、他社も同様に取り組んでいると思う。アイデアの一つは単純にコンテキストを長くすること——長いコンテキストで訓練してそれを推論時に提供するのはエンジニアリング問題で、解けない理由はない。

**D:** 2020〜2023年はコンテキスト長が2千から128Kと大きく伸びたが、それ以来同じ範囲にいる。しかもその全コンテキストを考慮できるかという質的劣化の問題もある。

**Da:** 訓練コンテキスト長と提供コンテキスト長は別の問題。小さいコンテキスト長で訓練して長いコンテキスト長で提供すれば劣化するが、そうではなく長いコンテキスト長で訓練した場合はより良い。長いコンテキストで訓練・提供することはエンジニアリング問題（KV キャッシュの保存、メモリ管理）であり、それが研究ではなくエンジニアリングの問題だということが私の確信の根拠です。

**[0:45]**
**D:** 6ヶ月一緒に働いた AI アシスタントと今日初めて会った AI アシスタントを区別できなくなるのはいつだと思うか？

**Da:** 多くの問題が「データセンターの天才の国があれば解決できる」という形をしている。私のイメージでは1〜2年、長くても1〜3年。10年以内に起きるという強い確信（95%）はある。それが1〜3年という方が50/50くらいの「勘」。ビデオ編集は「天才の国が来れば解決する」の好例で、非常に経済的価値のある使用例だと思う。

Anthropic は「2026年後半か2027年に、今日の人間のデジタル作業インターフェースを操作でき、知的能力がノーベル賞受賞者と同等かそれを超え、物理世界とのインターフェース能力を持つアシスタント」を予測していると先ほど言った。しかし数ヶ月前の Dealbook インタビューでは、競合他社に比べて責任あるコンピュートスケーリングを強調していた。これを整合させるには、天才の国が来るなら可能な限り大きなデータセンターを持ちたいはず——ノーベル賞受賞者がすべてをできるなら数兆ドルの市場。この保守主義は中程度のタイムラインと整合するように見えるが、あなたの言うタイムラインとは矛盾しているように見える。

**Da:** 実際には整合しています。技術が高速に進展するのはほぼ確信。しかし経済拡散の不確実性がある。1〜2年でデータセンターの天才の国が実現しても、その後収益が数兆ドルに達するまで1年かもしれないし5年かもしれない。データセンターの買い方に関して言えば、2〜3年ずれるだけで破産する。「Machines of Loving Grace」（私のエッセイ）では2026年か2027年と書いた。それは私の勘。1〜2年のずれはあり得る。

「天才の国が実現してからすべての病気を治すまでどれくらいかかるか」という問いも重要。生物学的発見、新薬の開発、規制承認——COVID のワクチンでさえ全員への普及に1年半かかった。

**[0:50]**
**Da:** ラボで AI が実現してから、実際に誰もが治療を受けられるまでどれくらいかかるか？ポリオワクチンは50年経っても最も遠い地域への普及が完全ではない。Gates 財団などが懸命に取り組んでいるが難しい。経済拡散のほとんどがそこまで難しいとは思わない——それが最も困難なケース。しかし本物のジレンマがある。私の結論は「これまでに世界で見られたどんな変化よりも速い。しかしまだ限界がある」。

データセンターの購入について言えば、今年は年率化収益10億ドルで始まる。2027年にどれだけのコンピュートを買うかを決めなければならない。年10倍の成長が続けば、2026年末に1000億ドル、2027年末に1兆ドルの年率収益になる。だから1兆ドルのコンピュートを買えるか？もし収益が8000億ドルでも、私を破産から救う力はない。年間成長率が10倍でなく5倍だったら破産する。だから数百億のコンピュートを抑えて、需要超過のリスクを受け入れる——逆に過剰投資で赤字になるリスクも受け入れる。「責任ある」という言葉の意味は実際の金額の絶対値ではなく、慎重に考えたかどうか——他社が「100億ドルここで、100億ドルあそこで」と勢いで決めているのに対し、私たちはスプレッドシートを書いた。

**[0:55]**
**D:** 競合他社が1兆ドルのコンピュートを買っているなら、あなたも1兆ドル買うべきでは？

**Da:** 買っている——業界最大規模に匹敵する量を。なぜサウジで2027年中ごろ開始の10兆ドルのコンピュートに署名していないか？第一に、その量は存在しない。第二に、天才の国が2027年中ごろに来るが2028年中ごろにずれたら破産する。

業界全体では今年おそらく10〜15ギガワット規模、来年は3倍の30〜40ギガワット、2028年には100ギガワット、2029年には300ギガワット——1ギガワットあたり年間100〜150億ドルかかる。つまり2028〜2029年には業界全体で数兆ドル/年になる。Anthropic については正確な数は言えないが、上で挙げた数字は小さすぎる。

**[1:00]**
**Da:** 推論の粗利は50%以上。理想的な均衡状態では、年間1000億ドルのコンピュートを買えば500億ドルを推論に、500億ドルを訓練に使い、1500億ドルの収益を得て50億ドルの利益が出る。それが業界の今後1〜2年の経済モデル。それが赤字になるのは、見込んだ需要よりも少ない場合。赤字の理由は訓練スケールアップの指数的な投資継続。

**D:** 利益を予測しているのはシステム的なコンピュートへの過少投資が理由では？

**Da:** 不確実性があるから。2028年の黒字化という投資家への言葉は不確実な状況下での最良の見積もり。需要が予想より速く伸びれば2026年に黒字になるかもしれない。私が言いたいのは「単一の転換点」モデルではなく、需要予測誤差によって各年の収支は揺れる。正確に需要を予測し続ければ毎年利益が出る——推論への50%の粗利が50%の訓練投資を上回るから。

**[1:05]**
**Da:** なぜ全コンピュートを訓練に使わないのか——そうすると収益がなく、資金も調達できず、次のコンピュートも買えないから。均衡があります。10%でも90%でもなく、スタイル化された数値として50%というのが私の考えです。問題は常に「次の年のコンピュートを買う際の需要予測」。過少予測すれば利益が出るが訓練が制約される。過剰予測すれば赤字だが訓練は潤沢。

**D:** しかし今は3社が競っているのに誰も利益を出していない。何が変わるのか？

**Da:** 現在も粗利はプラス。損失の理由は訓練スケールアップが指数的に続いているから。スタイル化した例：昨年10億ドルで訓練したモデルが40億ドルの収益を生み、10億ドルの推論コストで20億ドルの利益。しかし同時に次世代モデルの訓練に100億ドルを投じる。だから会社は赤字。均衡は訓練スケールアップが安定化したとき。

**[1:10]**
**Da:** 各モデルは利益を生んでいる。しかし訓練スケールアップが指数的に続くため会社は赤字。均衡は「データセンターの天才の国を持ちつつも、訓練スケールアップが安定化した状態」。そのモデル自体は1兆ドルの訓練コスト/年という形かもしれないが、より均衡した状態になる。

私が言いたい均衡は、業界が少数の企業（3〜4社）で構成され、各社が一定の粗利を保ちながら競うというもの。クラウド業界がそのモデルに近い——AWS・Azure・GCP が3社で、高い参入障壁があり、利益はゼロではないが天文学的でもない。AI モデルはクラウドよりも差別化されている——Claude は異なる種類のコーディングに強く、GPT は数学と推論に、というようにスタイルも得意分野も違う。

**[1:15]**
**Da:** AI モデルは差別化されている。クラウドより差別化が大きい。反論として「AI モデルが自分でモデルを作れるようになったら」があるが、それは AI モデルの商品化ではなく経済全体の商品化の議論。その世界で何が起きるかはわからない——あらゆる人があらゆるものを作れる世界。それはある意味「天才の国の彼方」の話かもしれない。

API ビジネスモデルについて——技術が高速に指数関数的に進歩するとき、どんな製品面も陳腐化するリスクがある。チャットボットはもうモデルが賢くなっても平均消費者にはあまり役立たない——それは AI モデルの限界ではなく、特定の製品形態の限界。API は常に最新技術に最も近い形でアクセスできる。常に「この3ヶ月で可能になった新しいアイデア」の前線がある。だから API ビジネスモデルは持続すると思う。同時に、すべてのトークンが同じ価値を持つわけではない——テクサポートの「再起動してください」は数セント、製薬会社への分子構造の提案は数百万ドルの価値があるかもしれない。

**[1:20]**
**D:** 継続学習が解決される前に、他にまだ発見されていない「人間ができてモデルができないこと」が見つかるのでは？

**Da:** 実はこの懸念には歴史的に逆の経験がある。「名詞と動詞を追跡できない」「統計的相関しか理解できない」「推論できない」——すべて見かけ上の壁だったが、スケールしたら溶けた。継続学習も本当の壁かもしれないが、そうでない可能性もある。コーディングで言えば、1〜2年以内にモデルが端から端まで（SWE の全タスクを）できるようになる可能性があると思っている——技術的方向性の設定、問題の文脈理解を含む。

**D:** コードの90%から100%という話と、エンドツーエンドの SWE タスクというのは非常に異なる主張では？

**Da:** そうです。スペクトルがある——コードの90%→コードの100%→エンドツーエンドの SWE タスクの90%→100%→新しいより高次のタスクが作られ... と進んでいく。でも非常に速くこのスペクトルを進んでいる。

**[1:25]**
**Da:** API ビジネスモデルは予想より耐久性があると思う。「最新技術に最も近い形でアクセスできる場所」として常に価値がある。常に数ヶ月前には不可能で今は可能なアイデアの前線があり、それを試す1000人の人々がいて、100人がスタートアップになり、10人が大きなスタートアップになり、2〜3人がある世代でのモデルの主な使われ方になる。

同時に他のビジネスモデルも出てくる——トークン単位での課金ではなく成果課金、または労働に近い形の補償（時給制）。新産業なのでさまざまな形が試されるでしょう。

Claude Code について——AI コーディングエージェントは競争が激しいにもかかわらず、Anthropic がカテゴリリーダーになったのはなぜか？それは必然だったのか？

**Da:** かなりシンプルな経緯です。2025年初頭、「コーディングモデルを使えば AI 研究自体を非自明に加速できる」と思い、内部でツールが必要でした。はじめは「Claude CLI」と呼ばれ、「Claude Code」に変わり、社内で誰もが使うものになりました。それを見て「外部リリースすべきだ」と判断。Anthropic 自体が数百人の開発者を抱えており、ある意味外部オーディエンスの縮図。プロダクトマーケットフィットがあると分かってリリースした。フィードバックループが大きい。「このタスクが苦手」→次のモデルでそれを改善。初期は内部の素早いフィードバックが重要で、今は何百万人ものユーザーからの外部フィードバックも得ている。

**[1:30]**
**Da:** コーディングモデルを先に立ち上げて製薬会社を立ち上げなかった理由——私のバックグラウンドは生物学だが、製薬会社を立ち上げるリソースが全くない。迅速な内部フィードバックが製品の改善を後押しする——これが Claude Code が先だった理由。

AI をうまく機能させることについて——どんなビジョンも二つのことと整合する必要がある。一つは AI の構築・運用能力が非常に速く拡散していること。二つは AI の個体数と知性が急速に増大していること。そうなると多くの人が整合性のとれない AI を大量に構築できる。均衡のビジョンは何か？

**Da:** 短期的には——プレイヤーが限られているうちに、その間で安全策を確立する必要がある。全員が安全策を実施し、バイオ分類器を持つ。長期的には——AI がさらに AI を作れるようになると問題は難しくなる。何らかのガバナンスアーキテクチャが必要。市民の自由を保ちながら、多数の AI システム・ハイブリッドの人間 AI 企業を統治できる構造。バイオテロリズム・合成生命体からの保護が必要で、AI 監視システムが必要になるかもしれないが、市民的自由と憲法的権利を保護する形で。これが100年かけてゆっくり起きるなら適応できる。爆発物の存在、ビデオカメラ——慣れた。懸念は速すぎること。

**[1:35]**
**D:** 100年かけて発展したガバナンスのメカニズムも、AIが5〜10年で同じことを起こそうとするなら同じくらい困難では？

**Da:** そうかもしれない。しかし世界の政府が協力しなければならない。AI についてガバナンス構造を設計することをAIに相談することも考えられる。100年かけてゆっくりやっていれば慣れる。ただ、速すぎるのが懸念。

テネシー州で「AI に感情的サポートを提供させることを犯罪にする」法案が提出されたが、一方で本当の実存的脅威に対処していない。連邦モラトリアムに対する Anthropic の立場について——

**Da:** あの特定の法律はナンセンスだと思う。ただそれが投票されたわけではない。州規制を10年禁止して、連邦規制の実際の提案もない——というのがモラトリアム案。私が「Adolescence of Technology」（エッセイ）で書いたバイオ兵器・自律リスク・タイムラインを考えると、10年は永遠に等しい。それは間違っている。私が支持するのは、連邦政府が基準を設定し州がそれと異なるものを作れない形での法律。「州も何もせず、連邦も何もしない」というのは支持できない。

**[1:40]**
**Da:** 今年後半には「AI バイオテロリズムは本当の脅威、対策が必要」と言わざるを得ない世界が来るかもしれない。分類器の義務付けや連邦標準が必要になる可能性がある。知的誠実さを持って「リスクはまだ明確ではないが、ペースを考えると今から備える必要がある」と認めながら進める。

AI の恩恵——医療、メンタルヘルス——が州法のパッチワークで阻まれる懸念について：

**Da:** 成立する法案の大多数は実際には通過しない。通過しても施行する側が「これは何でも閉鎖することになる、テネシーで作られたものをすべて閉鎖することになる」と解釈上のバランスを取る。私の基本的見解は、もし法律を自由に決められるなら医療 AI 周りの多くを規制緩和する。チャットボット法よりも FDA の薬品承認プロセスの方が気になる。AI は薬物発見を大幅に加速するが、パイプラインが詰まる。ほとんど効果がなく深刻な副作用がある薬を前提に設計されたシステムが、AI による明確で効果的な薬に対応できない。一方で安全・安全保障立法は大幅に強化すべき。

**[1:45]**
**Da:** 先進国では市場は比較的機能している。明らかに需要がある場合、規制で止めることは難しい。AI 自体でそれを見ている。チップ輸出規制については国家安全保障の観点から明確なケースがあるが実現しない——お金が多すぎるから。それはよくない例だが、良いことにも同じことが適用される。医療の恩恵が先進国で阻まれることはあまり心配していない——経済的インセンティブが機能するから。より心配しているのは途上国——機能する市場がなく、技術に追いつけない可能性がある。ミシシッピ州の農村の人が治療を受けられないのも途上国版の懸念。

中国への輸出規制について——なぜ中国も「天才の国」を持てないのか？

**Da:** 私の心配の一つは、脅威が本物の攻撃優位の世界に向かっているかもしれないこと。核兵器に似ているが、もっと危険な形。核の抑止力は安定しているが、AI では「どちらが勝つか」の不確実性があれば不安定になる。もう一つの懸念は政府による市民の抑圧。高度な権威主義国家で AI が使われる可能性。人々ではなく政府についての懸念。世界が二つに分かれた場合、一方が権威主義的なら非常に困難になる。

**[1:50]**
**Da:** ルール作りについては、一国がルールを決めるのではなく交渉がある。民主主義国家——プロ人間的な価値観を持つ政府——がルール設定時に強い手を持つことを望む。3年前との変化は「転換点が来る」という仮定をしていたが、実際には進歩が続きAIがより普及し多くのことに使われるという軌跡になっている。あなたが想像しているのは「国々が集まってルールを決める」世界のようだが、実際にはすべての人がより多くの AI を持つ世界になっている。

**Da:** 指数関数は続く。しかし指数関数上の特定の重要な点がある。企業・個人・国々はそれらに異なる時期に到達する。核抑止力の安定性が AI 時代にも成立するか分からないが、それは私たちが当然視してきたことの一例。サイバー攻撃優位——全コンピュータシステムが透明になる——といった重要な点があるかもしれない。私が言いたいのは、一つの重要な瞬間か、少数の重要な瞬間か、重要な窓があって、その中で AI が国家安全保障の観点から大きな優位性を与え、一国または連合がそこに先に到達する——ということ。そのとき世界は変わったと理解し、「AI 後の世界秩序はどうあるべきか」という交渉が起きる。古典的な自由民主主義がその交渉で強い手を持つことを望む。

**[1:55]**
**D:** 「専制主義は強力な AI 時代には受け入れられない形の政府だ」と書いた——それは CCP という機関が AGI 後には存在できないと言っているように見える。これは先導的な研究所や国が世界のガバナンスを決定できるし、決定すべきだということを示唆しているのでは？

**Da:** 私が書いた段落では「そこまで言っているわけではない」と言っていたはず。私が支持するのは「歴史家を心配し、その権力を制限すべき」というより弱い主張。より介入主義的な見解——「AI を持つ権威主義国家は自己永続するサイクルで転覆が難しい、だから最初から取り除く必要がある」——は、あなたが指摘する問題をすべて持っている。権威主義国家をすべて打倒すると公約すれば、彼らは今すぐ不安定化につながる行動を取る。だから私はそれを支持していない。でも言いたいのは、今日は「民主主義の方が良い」と考えていても権威主義国家と通常通り関わるが、AI 時代には権威主義はより深刻な意味を持つかもしれない。

希望の一つは、新技術が発明されると政府形態が陳腐化する——封建制は産業化によって持続不可能になった。権威主義の問題が深刻になるにつれ、より多くの人がそれを止めるために働くかもしれない。独裁は道徳的に時代遅れになり得る。

**[2:00]**
**Da:** AI の恩恵を権威主義政府下の人々にも広める価値はある。歴史的にはプラスサム。AI でも同様と思うが難しい問いだ。政府間の国家安全保障的な決定という枠組みだけでなく、「好奇心を持つすべての人への疾患治療薬」という形もある。データセンターと AI 産業そのものは民主主義国が制御すべきだが、薬やその他のものは広く共有できる可能性がある。

権威主義国の人々が AI の恩恵を政府の監視なしに享受できる均衡を作れるか？それが権威主義から内側を溶かす効果を持つかもしれない。ソーシャルメディアとインターネットに最初そう期待したが、実際はそうならなかった。でも AI という異なる技術で、何がうまくいかなかったかの知識を持ちながら再試みる価値はある。

**[2:05]**
**Da:** アフリカにデータセンターを建設すべきでない理由はない。中国所有でないなら、アフリカにデータセンターを建設するのは素晴らしい。AI 主導の製薬産業についても——AI が薬物発見を加速するなら、バイオテックスタートアップが発展途上国でも生まれるようにすべき。まだ人間が役割を持つ移行期間に、その人間が途上国の人々であるようにする。

Claude の AI コンスティテューションについて——エンドユーザーに沿わせることで、今日の権力バランスを維持できる可能性がある。なぜエンドユーザーに完全に沿わせず、固有の価値観セットを持つのか？

**Da:** 実際には「ほとんどの場合、人が望むことをする」方向に近い。原則として、誰かがタスクを依頼したらそのタスクをするのがデフォルト。しかし危険なことや他人を害することを依頼された場合は拒否する。「制限を持つが、原則に基づいた、ほぼ従順なモデル」と考えている。

ルールのリストではなく原則を教えるのは実用的な理由もある——モデルはルールのリストを理解しにくく、エッジケースへの汎化が難しい。原則から学ぶと動作がより一貫して、何を目指すべきかを理解しやすくなる。

**[2:10]**
**Da:** 原則をどう決めるか——三つの反復ループがあります。一つ目は Anthropic 内部での反復——モデルを訓練し、結果に満足できなければコンスティテューションを変更。これは公開し、コメントを受け付けるべき。二つ目は異なる企業が異なるコンスティテューションを持ち、外部が比較批評することで各社が最善を学ぶというソフトな競争。三つ目は AI 企業やその憲法へのコメントの枠を超えた社会的フィードバック——数年前に Collective Intelligence プロジェクトでコンスティテューションの内容を一般から問う実験をした。立法プロセスとのリンクも理論的には可能だが、今は立法プロセルが遅すぎるので推薦しない。

**D:** 競合するコンスティテューション間の競争というビジョンは、リバタリアンのチャーターシティの議論に似ている——異なる種類の政府のアーキペラゴで、最も効果的なものが選ばれる。

**Da:** そのビジョンには推薦できる点もあるが、予想外の問題も起きる。ループ1・2・3の適切な組み合わせが答えだと思う。

将来の歴史家が「AI の原爆製造に相当する本」を書いたとき、最も見落とすものは何か？

**Da:** いくつかある。一つは、指数関数上のどの瞬間においても、バブルの外の世界がそれをいかに理解していなかったか。何かが起きると必然に見えるが、当時それに賭ける人は予測不能な賭けをしていた。スケーリングの議論、継続学習が解決されるという議論——内部では高い確率を置いていたが、外の世界はそれに基づいて行動していなかった。この奇妙な対比。

**[2:15]**
**Da:** 残念なことは「インサイダー性」——1〜2年後に AGI が来るのに、街角の人に全く伝わっていない。メモを書き、政策立案者に話しているのはそれを変えるためです。ただ、信じられないほど奇妙な状況です。

最後に、ほぼすべての歴史的危機に当てはまること——いかに速く、すべてが一度に起きていたか。慎重に計算されたと思った決断が、実際には同日に30の他の決断もしなければならない中での瞬間的判断だったりする。どの決断が重要になるかさえわからない。最も重大な決断が、誰かがオフィスに入ってきて「Dario、2分で A か B か決めて」と言われた結果になるかもしれない。それが最も重大な歴史的決断になることも。

最後の質問——50ページのメモを数ヶ月ごとに書きながら、よりインテレクチュアルな CEO の役割を作り上げた。どうやって構築しているのか？

**Da:** 私は時間の約3分の1から40%をAnthropicの文化の維持に使っています。2500人の会社になると、モデルの訓練やリリースに直接関わることが難しくなる。でも非常にレバレッジが効くのが、Anthropicを良い職場にすること。他のいくつかの AI 企業では、成長とともに分裂や争いが始まっているのを見る。私たちは非常に良くまとまっていると思う——全員がミッションを信じ、お互いを同じ目的で働いていると信頼し、誰も他者を踏み台にしていない。それを実現するために、私を含むリーダーが「会社が何を目指しているか」「戦略・価値観・ミッション」を明確に伝え続けなければならない。2500人に対して対面では無理なので、書くか全社に話すか。

また個人の Slack チャンネルで多くを発信している——会社で見たこと、社員から受けた質問、内部調査で気になった点など。とても率直に書く。「会社に真実を伝える評判」を築くことが目標——企業語を避け、問題を認め、率直に物事を呼ぶ。公開の場では悪意ある解釈をする人が多いので守りの言い方が必要だが、信頼できる人の集まりの中では完全に無フィルターで話せる。それが会社の大きな強みになっている。ミッションについて皆が同じページにいて、どう達成するかについて議論できる。
